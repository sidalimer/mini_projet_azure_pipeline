  connection: 'BLOB_CONNECTION_STRING',
  extraOutputs: [signalRMessages],
  handler: async (blob, context) => {
    const { documentId, fileName } = context.triggerMetadata;

    // 1. garde-fous : fichier vide / doc inexistant -> ERROR
    // 2. patch UPLOADED  (Cosmos)
    // 3. sendMessage(Service Bus, { documentId, fileName, blobName, size })
    // 4. patch QUEUED    (Cosmos)
    // 5. push notifications SignalR : UPLOADED + QUEUED
  }
});

Le pattern {documentId}/{fileName} permet d'extraire l'ID directement depuis le chemin du blob — aucun appel supplémentaire requis.

Service Bus + Dead Letter Queue

Config ARM


{
  "maxDeliveryCount": 3,
  "deadLetteringOnMessageExpiration": true,
  "defaultMessageTimeToLive": "P14D",
  "lockDuration": "PT1M"
}

Mécanique

  • Notre code throw en cas d'erreur
  • Service Bus relivre 3 fois
  • 4ème échec → message en $DeadLetterQueue automatiquement
  • Zéro code custom pour la DLQ : c'est le broker qui gère

Queue : documents-queue

Retourne uniquement un tableau JSON de chaînes.

Réponse


["cv", "rh", "azure", "cloud", "pdf"]

Pipeline

  1. Validation du message (sinon throw → DLQ)
  2. Cosmos : PROCESSING
  3. SignalR : « Traitement IA en cours »
  4. Azure OpenAI (GPT-4o-mini)
  5. Fallback règles si échec IA ✓ autorisé
  6. Cosmos : PROCESSED + tags
  7. SignalR : « Tagging terminé » + tags

Function #3 — DLQ Alert


app.serviceBusQueue('dlqAlert', {
  queueName: '%SERVICE_BUS_QUEUE%/$DeadLetterQueue',
  connection: 'SERVICE_BUS_CONNECTION_STRING',
  extraOutputs: [signalRMessages],
  handler: async (message, context) => {
    const reason = context.triggerMetadata?.deadLetterReason;
    const desc   = context.triggerMetadata?.deadLetterErrorDescription;

    await patchDocument(documentId, {
      status: 'ERROR',
      errorMessage: `${reason}: ${desc}`,
      errorAt: new Date().toISOString()
    });

    context.extraOutputs.set(signalRMessages, buildSignalRMessage({
      documentId, status: 'ERROR', message: 'Erreur de traitement (DLQ)'
    }));
  }
});

Filet de sécurité ultime : aucun message ne tombe dans le vide. Cosmos est mis à jour avec le motif détaillé du broker, l'UI affiche un badge rouge en temps réel.

Côté React


const conn = new HubConnectionBuilder()
  .withUrl(`${FN_URL}/api`)
  .withAutomaticReconnect()
  .build();

conn.on('documentUpdate', (evt) => {
  if (evt.documentId !== currentId) return;
  setJob(prev => ({
    ...prev,
    status: evt.status,
    tags:   evt.tags ?? prev.tags
  }));
});

await conn.start();

SignalR Service en mode Serverless (requis pour le binding) — Hub documents, event documentUpdate.
